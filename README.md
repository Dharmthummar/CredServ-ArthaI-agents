### Demo: See the verifier catch a math error (no LLM needed)

```bash
python demo_failure.py
```

This shows — without any model running — how the deterministic verification engine:
1. Detects 4 balance mismatches in the LLM's first output
2. Injects the failure reason into a retry prompt
3. Confirms the corrected output passes

### Key design decisions

| Decision | Rationale |
|---|---|
| Image-only input to VLM | Eliminates prompt injection via embedded PDF text |
| JSON-only system prompt | No regex; schema validation via `json.loads` |
| `Decimal` arithmetic in verifier | Avoids float rounding errors |
| Retry with failure reason | Model can self-correct with specific feedback |
| Ollama default backend | Zero data egress — PII never leaves the machine |

### State Transition Diagram

```
PENDING_D15 (D-15)
    │  timeline trigger
    ▼
REMINDER_SENT_D7 ── SMS + Email sent
    │  timeline trigger
    ▼
FINAL_REMINDER_D1 ── Push notification
    │  check payment gateway
    ▼
DUE_DATE_CHECK ──[paid]──────────────► CLOSED_PAID ✅
    │ [unpaid]
    ▼
DELINQUENT_D1 ──[paid]───────────────► CLOSED_PAID ✅
    │ [unpaid]
    ▼
DELINQUENT_D3 (AI Voice Call)
    ├──[dispute]────────────────────► CLOSED_DISPUTED ⚠️  (human review)
    ├──[promised but unconfirmed]──► ESCALATED_HUMAN 👤
    └──[paid confirmed]────────────► CLOSED_PAID ✅
```

**Simulate paid path:** Set `mock_check_payment_status` → `return True`  
**Simulate dispute path:** Set `simulate_dispute = True` (default)
